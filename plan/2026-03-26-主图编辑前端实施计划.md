# 主图编辑 SaaS 前端重构实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把当前前端从“内部演示工具壳”重构为“内部试运营 SaaS 作图平台”，落地登录页、App Shell、Dashboard、聊天作图、批量作图、素材库、生成历史和设置页，同时保留现有聊天生成与 PSD 批量工作流主逻辑。

**Architecture:** 保留现有 React + Vite + TypeScript 单页应用和现有 API 契约，在顶层新增试运营账号态与平台级资产/活动上下文，再用新的 App Shell 包裹聊天、批量与资产模块。视觉层统一改为纯白、简洁、克制、接近 ChatGPT 工具页的 SaaS 风格；业务层只补前端状态与路由重组，不在本轮接入真实认证、计费或订阅后端。

**Tech Stack:** React 18, React Router DOM 7, TypeScript, Vite, 现有 `frontend/src/styles.css` 全局样式、现有 `frontend/src/api.ts` 与 `frontend/src/context.tsx`

---

## 实施边界

- 本计划只做前端重构，不接真实登录、注册、计费、额度后端。
- 用户、额度、最近活动、套餐信息使用前端假数据或本地持久化。
- 保留现有聊天生成 API、素材库 API、PSD 模板上传/替换组/生成主流程。
- 不在本轮引入新的测试框架；验证以 `npm run build` + 手动路由核验为主。
- 如某步需要删除旧页面文件，仅在确认新路由已完全替代后再删除。
